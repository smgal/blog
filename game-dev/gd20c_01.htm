서론

    새 하드를 사서 이런 저런 소스들을 정리하다가 이따위 게임이 방치된 채로 하드 디스크의 용량을 차지하더군요. 그래서 이곳 자료실로 백업을 합니다.
     
    몇 년전인가 쯤에 저의 게임 20번째를 자축하는 의미에서 만들었던 걸로 기억합니다.
    내용은 10년 전에 만들었던 Apple 용 자작 퍼즐 게임의 리메이크입니다.

게임 설명

    실행 방식은 쿼터뷰 real-time 입니다.
    마우스 왼쪽 버튼으로 이동 위치를 선택 할 수 있으며 키보드로도 위치를 이동 시킬 수가 있습니다.
     
    이 게임의 목적은 일단 IQ 를 테스트 하기 위한 퍼즐이며, 클리어 조건은 각 스테이지마다 하나 씩 있는 대변(shit : 앙증맞고 귀엽게 숨고르기 하는 물체)을 무지 막지한 벽돌로 압사를 시키거나, 온갖 난관을 다 헤치고 무사히 변기에 진입을 하면 됩니다.
     
    벽돌은 밀면 밀리는 방식이나 둘 이상이 붙어 있으면 힘이 부쳐서 밀지 못합니다. 또한 바닥에 있는 웅덩이는 벽돌로 메울 수가 있습니다.
     
    더 자세한 것은 자동 해답 찾기를 실행해 보면 알 겁니다.

게임 정식 등록

    이 게임은 '등록 외침' 버전입니다.
    게임을 실행시킨 상태에서 키보드에서 손을 떼고
     
    " 나는 정식 사용자다 "
     
    라고 크게 외치며 만세 삼창을 한 후에 '등록' 버튼을 누르면 그때부터 정식 사용자가 됩니다.
    정식 사용자에 대한 특전은 자동 해답 찾기 기능이 추가되어서 도저히 안풀리는 경우에는 컴퓨터가 해답을 시연해 줍니다..
     
    모르는 점이 있으셔도 저에게 메일 주시면 안됩니다. 업무 시간에 답장 쓰다가 회사에서 짤리면 당신이 책임을 질 겁니까 ?

소스有

위의 게임의 소스입니다.
 
Delphi2 지원 당시에 만들었던 소스이기 때문에 Activex unit 가 아닌 OLE2 unit 을 사용합니다.
libDelphi2 에 있는 OLE2.dcu를 lib로 copy하시면 Delphi 5 등에서 잘 컴파일 될겁니다..
 
소스를 통해 배울 점
 
1. 단 하나의 EXE 로 게임을 만드는 방법
2. Direct X 로 게임 만드는 법
3. 스타크같은 자동 길찾기 알고리즘
4. 쿼터뷰 출력 알고리즘
5. 쿼터뷰 좌표 변환 알고리즘
6. 프레임 동기화
 
등등이 있고 ...
 
소스 내의 플래그를 조절하면
 
1. 맵 에디터 모드 진입
2. 창모드 게임
3. 프레임 비동기화
4. 퍼포먼스 출력
 
등등을 바꾸어 가며 테스트 할 수 있습니다.
 
소스에 대한 질문은 받지 않으며, 상업적으로 쓰이지 않는 한, 모든 것이 당신 것입니다.
